Suika3 系统需求规格
===========================================

## 1. 概述

Suika3 是一个专为视觉小说和 2D 游戏优化的高性能脚本运行时。它提供了一个多层级的领域特定语言环境，旨在平衡易用性与专业级的可扩展性。

## 2. 核心组件（领域特定语言栈）

Suika3 通过四种专用语言为创作者赋能：

- NovelML（基于标签的领域特定语言）：一种直观的标记语言，使用 `[]` 标签，用于快速开发视觉小说剧本。

- Anime（动画领域特定语言）：一个基于层的光栅图像动画的专用系统，用于控制仿射变换序列。

- GUI（用户界面/用户体验领域特定语言）：一套灵活的工具集，用于构建带有按钮的交互式屏幕，并针对视觉小说的需求进行了优化。

- Ray（通用脚本）：一种拥有视觉小说 API 的强大脚本语言。
    - 定制化：定义定制的 NovelML 标签。
    - 性能：在 PC 上进行即时编译以实现快速迭代；提前编译为原生二进制文件以符合 iOS 规范。
    - 底层访问：直接挂钩到 Suika3 核心 C API。

## 3. 关键目标与设计哲学

- 移动端优先体验：基于智能手机是主要计算设备的理念而设计。它摒弃了以 PC 为中心的用户界面/用户体验，转而追求原生的移动设备感觉。

- 应用商店发布兼容性：通过提前编译和响应式设计，完全符合 iOS/Android 应用商店的政策。

- 高可移植性：
    - 第一梯队：iOS、Android、HarmonyOS NEXT、Windows、macOS、Linux
    - 第二梯队：游戏主机
    - 第三梯队：Chromebook、Wasm（WebAssembly）

- 超越视觉小说：虽然以视觉小说为核心，但其底层的 2D 基础允许进行类型融合（例如，视觉小说 + 角色扮演游戏或动作游戏）。

## 4. 范围外 / 限制

为了保持可移植性和性能，Suika3 明确排除了以下内容：

- PC 独占功能：Suika3 不是旧式 PC 独占视觉小说引擎的替代品。

- 全规模 Web 部署：Wasm 移植版旨在用于演示，而非主要分发渠道。

- 3D 图形：目前专注于 2D（计划在未来结合 AI 驱动的资产生成来支持 3D）。

- 专有中间件：不支持像 Live2D 这样的封闭技术，以确保引擎的最大可移植性。
